## CRITICAL Issues (P0 — Fix Immediately)

### 1. `next.config.mjs` Silently Overrides Real Config

**Severity: CRITICAL — Production security headers are NOT being applied**

Two config files exist:

- `next.config.js` (179 lines) — The GOOD config with security headers, CSP, Sentry, transpilePackages, image optimization
- `next.config.mjs` (11 lines) — Contains `ignoreBuildErrors: true` and `images: { unoptimized: true }`

**Next.js loads `.mjs` over `.js` by priority.** This means:

- ALL security headers from `next.config.js` are **ignored in production**
- TypeScript errors are **silently hidden** during builds
- Image optimization is **disabled**
- Sentry error tracking is **not configured**
- `transpilePackages` for ESM modules is **missing** (potential runtime crashes on Vercel)
- CSP (Content Security Policy) is **not being served**

**Fix:** Delete `next.config.mjs`.

### 2. `postcss.config.mjs` Overrides `postcss.config.js`

Same pattern:

- `postcss.config.js` — Uses `tailwindcss` + `autoprefixer` (correct)
- `postcss.config.mjs` — Uses `@tailwindcss/postcss` which **isn't installed**

**Fix:** Delete `postcss.config.mjs`.

### 3. Dependency Conflict Blocks Clean Install

`npm install` fails due to `@composio/anthropic@0.6.4` requiring `@anthropic-ai/sdk@^0.52.0` but `^0.71.2` is installed. Currently requires `--legacy-peer-deps` workaround.

**Fix:** Resolve the peer dependency conflict properly.

### 4. `xlsx` Package Has Unfixed High-Severity Vulnerabilities

- Prototype Pollution (GHSA-4r6h-8v6p-xvw6)
- ReDoS (GHSA-5pgg-2g8v-p4x9)
- No fix available from maintainer

**Fix:** Replace all `xlsx` usage with `exceljs` (already a dependency).
